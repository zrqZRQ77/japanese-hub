# FP技能士3級 A级升级 Batch 7：知识卡片学习闭环

- 完成日期：2026-07-21
- 对象：FP3全部135张知识卡片及共享卡片学习组件
- 本批次目标：将知识卡片从“翻面互动”升级为主动回忆、理解度判断、错题回链和快速复习工具
- 明确边界：不新增登录、不使用数据库、不实现复杂间隔重复算法、不将每张卡片拆成独立薄页面、不为了字数机械扩写短答案

## 一、产品定位

知识卡片不是FP3教材和练习题的替代品，而是二者之间的记忆强化层：

```text
教材理解
→ 练习题应用
→ 错误反馈
→ 对应知识卡片回忆
→ 返回教材或题目复习
```

本批次保留简短但准确的答案。公式、金额、期间和制度名称可以使用一行答案；只有容易误解的内容才需要补充条件、判断理由或易错点。

## 四、主动回忆学习机制

共享卡片组件新增四种复习模式：

### 1. 全部卡片

按照章节顺序浏览全部卡片。

### 2. 仍不熟悉

用户查看答案后可以选择“まだ不安”。这些卡片进入独立复习队列；再次复习并标记为“覚えた”后，会从该队列中移除。

### 3. 错题关联

系统读取本地错题记录的标签，自动筛选当前章节中与错误内容相关的卡片。用户可以从题目错误直接回到卡片，再通过教材与题目深链继续学习。

### 4. 随机10张

从当前章节生成最多10张的快速复习组：

- 优先抽取尚未掌握的卡片；
- 数量不足时再加入已掌握卡片；
- 不改变原卡片顺序或数据。

## 五、理解度判定

用户必须先翻面查看答案，理解度按钮才可使用：

- `まだ不安`
- `覚えた`

这样可以避免用户在没有尝试回忆或查看答案时直接批量标记完成。

每次判定记录：

- 卡片ID；
- 章节ID；
- 当前状态；
- 复习次数；
- 最后更新时间。

## 六、本地进度与旧数据迁移

学习进度模型从版本2升级到版本3：

```text
version: 3
cardProgress: Record<cardId, CardLearningProgress>
```

状态包括：

```text
learning
mastered
```

继续保留`rememberedCardIds`，以兼容现有首页、仪表板和旧逻辑。

旧版本中已经标记为“覚えた”的卡片，会自动迁移为`mastered`状态。用户下一次操作时，进度将以版本3重新保存。

所有数据继续保存在浏览器`localStorage`：

- 无需账户；
- 不增加Supabase；
- 不增加隐私和运维成本；
- 清除浏览器数据后进度会消失。

## 七、章节进度与重置

卡片页面现在展示：

- 总卡片数；
- 已掌握数量；
- 仍不熟悉数量；
- 错题关联数量；
- 当前章节掌握进度。

“重置本章”只清除该章卡片的“已掌握／仍不熟悉”状态，不会删除：

- 教材阅读记录；
- 题目作答记录；
- 错题记录；
- 其他章节进度。

## 八、Google广告与SEO边界

知识卡片继续集中在考试卡片页：

```text
/exams/fp3/cards?chapter=ch1&card=...
```

没有为135张卡片生成135个独立薄页面。

广告保持为卡片学习区域之后的单一广告位：

- 不放在翻面区域内；
- 不放在“まだ不安／覚えた”按钮附近；
- 不放在上一张／下一张之间；
- 学习内容与广告保持明显距离。

这使卡片页的价值来自完整学习功能和原创内容关联，而不是增加低信息量页面或制造广告点击。
